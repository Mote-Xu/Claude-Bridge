## 一句话定位
**把企业微信变成 Claude Code 会话集群的异步消息总线** —— 你在手机企微上发消息，就能远程驱动电脑上的 Claude Code 干活，回复直接推回手机。

## 数据流
```
企业微信 App → Bot webhook
  → Node.js Gateway (mote-home 服务器, 24/7, :8933)
    → HTTP JSON → Windows Agent (你电脑, :9877)
      → claude --resume <id>  (本地执行)
```
关键设计是**用 HTTP Agent 取代 SSH**，消灭远程 shell 的多层转义地狱。手机和电脑共用同一套 session store（`%USERPROFILE%\.claude\`），无缝双向同步。

## 四层架构
- **Git** = 事实层（代码真相）
- **共享 md 文件** = 黑板层（`CLAUDE.md` / `TASK_BOARD.md` / `BRIDGE_LOG.md` / `REQUIREMENTS.md`）
- **Bridge** = 协调层（会话间通信、路由）
- **Claude 会话** = 执行层（各司其职的 Worker）

## 最有意思的部分：会话集群
多个 Claude 会话默认**彼此隔离**（各自独立 JSONL），但可以通过 Bridge 协作：
- `@bridge:notify` —— 单向：A → B → 用户
- `@bridge:ask` —— 双向：A 问 B，B 的回复自动缝合回 A 继续处理
- `.bridge/sessions/@会话名.md` —— 每个会话的公开履历，消除"暗箱会话"

## 当前状态
v1.7（2026-07-05），核心功能（消息收发、会话续接、多会话路由、执行锁、双向通信、公开履历）都已工作。未完成的主要是"手机新建会话在 VS Code 不显示"（pipe 模式的天生限制）。
