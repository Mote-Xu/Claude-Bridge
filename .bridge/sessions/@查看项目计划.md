## 项目定位

**手机（Telegram）是电脑前 Claude Code 的"第二屏幕"**——在手机上无缝续接电脑上已有的 Claude Code 会话，回到电脑继续。

## 架构

```
Telegram Bot API → Node.js Gateway (mote-home, 24/7)
                    → Tailscale SSH → WSL2 tmux → Claude Code（唯一 AI 执行体）
```

Gateway 不做 AI 推理，只做三件事：**消息路由、tmux 会话管理、SQLite 任务队列**。~300 行代码。

## v1 核心功能（6 项）

| 优先级 | 功能 |
|------|------|
| P0 | **历史会话无缝续接** — `claude --resume` 恢复，双向同步 |
| P0 | **多会话并行** — 多个 tmux session 同时跑，`/cc` 切换 |
| P0 | **交互式控制** — tmux send-keys 喂入，`/stop` 中断，弹窗盲操 |
| P0 | **流式输出** — 管道文件 tail 实时推送，非轮询 |
| P0 | **离线队列** — 笔记本关机时排队，上线后自动恢复 |
| P1 | **会话生命周期** — 首次自动创建，idle 24h 提醒清理 |

## 关键决策

- ❌ **不用 OpenClaw**（杀鸡用牛刀，避免双 AI 竞争）→ 改用 ~300 行 Node.js Gateway
- ✅ Telegram Bot（v1）替代 CF Tunnel 暴露端口
- ✅ tmux 持久会话而非 `claude -p` 一次性模式
- ✅ JSON Schema 结构化协议防命令注入
- ✅ chat_id 白名单 + audit 日志
